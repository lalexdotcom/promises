### Task 4.4: Verify Universal Build Configuration (NPM-04 - Part A: Config)

**Objective:** Ensure `rslib.config.ts` targets both Node.js 18+ and browsers with single universal build.

**Current State:**
- `rslib.config.ts` has only:
  ```typescript
  lib: [
    {
      format: 'esm',
      syntax: ['node 18'],
      dts: true,
    },
  ],
  ```
- Only ESM, no CommonJS (CJS)
- Issue: Browser users typically expect CJS or dual build

**Actions:**
1. Add CJS format to multi-format build:
   ```typescript
   export default defineConfig({
     lib: [
       {
         format: 'cjs',
         syntax: 'es2020',
         dts: false, // CJS shares .d.ts from ESM
       },
       {
         format: 'esm',
         syntax: ['node 18'],
         dts: true,
       },
     ],
   });
   ```

2. Rationale:
   - CJS (`dist/index.cjs`) for Node.js and bundlers that require CommonJS
   - ESM (`dist/index.js`) for modern Node.js, browsers, and ES modules
   - Both target only universal globals (Promise, setTimeout, console, standard library)
   - DTS only generated once (for ESM, shared by both)

3. Update `package.json` exports to point to both:
   ```json
   "exports": {
     ".": {
       "types": "./dist/index.d.ts",
       "require": "./dist/index.cjs",
       "import": "./dist/index.js"
     }
   }
   ```

4. Build and verify output:
   ```bash
   pnpm run build
   ls -lah dist/
   ```
   - Expected: `index.cjs`, `index.js`, `index.d.ts`, and `.map` files

**Verification Steps:**
- [ ] `rslib.config.ts` includes both `format: 'cjs'` and `format: 'esm'`
- [ ] `dts: true` only on ESM config (DTS generation)
- [ ] `package.json` exports includes `require` and `import` conditions
- [ ] Build succeeds: `pnpm run build`
- [ ] `dist/index.cjs` and `dist/index.js` both exist and are non-empty
- [ ] `dist/index.d.ts` exists (TypeScript definitions)

**Success Criteria:** Build produces both CJS and ESM outputs, ready for universal distribution.
