- [x] 01-02: Dead Code Removal + Type Improvements (BUG-03, BUG-04, TYPES-01/02/03)

**Requirements Completed**:
- ✅ **BUG-01**: Fixed inverted timeout guard in `runNext()` — `!Number.isNaN(timeout) && timeout > 0` now correctly fires per-promise timeouts
- ✅ **BUG-02**: Fixed `timeout()` in `utils.ts` — late resolution now silently ignored, wrapped promise rejections correctly propagated
- ✅ **BUG-03**: Removed duplicate `pending` getter (identical to `waiting`, not in interface)
- ✅ **BUG-04**: Removed commented dead code (`this.#emit('next')` calls)
- ✅ **TYPES-01**: `pool.parallel()` infers `Promise<[T1, T2]>` for heterogeneous, `Promise<T[]>` for homogeneous via overloads
- ✅ **TYPES-02**: `pool.serial()` uses same tuple/array inference as TYPES-01
- ✅ **TYPES-03**: `PromisePool` interface compiles with zero `any` on getters; all private fields use `unknown`

**Verification**: Passed (2026-03-23) — All 5 success criteria verified: timeout guard fixed, late-resolution silenced, parallel/serial type inference correct, zero `any` types, dead code removed.

### Key Decisions

- **Bugfix Strategy**: Three bugs (BUG-01, BUG-02, BUG-03) required minimal code changes (<5 lines each); one (BUG-04) was about removing misleading comments
- **Type System**: Upgraded pool.parallel/serial to use TypeScript function overloads for tuple inference, matching Promise.all() pattern — improves IDE DX significantly
- **Test Design**: Empirical concurrency testing (tracking simultaneous runners) rather than setTimeout-based assertions — more reliable and faster
- **Documentation Scope**: Focused on JSDoc (IDE hover tooltips) + README (onboarding + API reference) + inline WHY-comments on scheduler logic — all three needed for effective learning
- **Build Target**: Universal (Node 18+ and browsers) via Rslib with no platform-specific code — single source, dual output, full type definitions included
