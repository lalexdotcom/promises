## Naming Conventions

- **Files**: `camelCase.ts` (e.g., `pool.ts`, `utils.ts`)
- **Test files**: `<name>.test.ts` pattern in `tests/` directory
- **Classes**: PascalCase (e.g., `PromisePoolImpl`, `PoolErrorImpl`, `TimeoutError`)
- **Interfaces**: PascalCase (e.g., `PromisePool`, `PoolError`)
- **Functions/exports**: camelCase (e.g., `pool`, `wait`, `timeout`, `unsync`, `slice`)
- **Type aliases**: PascalCase (e.g., `PromiseFunction`, `QueuedPromise`, `PoolOptions`)
- **Constants**: SCREAMING_SNAKE_CASE (e.g., `DEFAULT_CONCURRENCY`, `DEFAULT_NAME`, `VERBOSE_LEVELS`)
- **Private fields**: `#` prefix (ES2022 private fields syntax)
- **Private methods**: `private` keyword

## Module Resolution

- Uses `"moduleResolution": "bundler"` in `tsconfig.json`
- Imports use relative paths with `.ts` extensions (allowed via `allowImportingTsExtensions`)
- No path aliases configured
